Engineering Intern.

Summer or part-time during the school year. Ship real features to real customers, not a walled-off side project. Paid, mentored, and credited on the commit history.

← All roles

The role.

Internships at small companies are either the best thing that happens to an early engineer's career or a summer of fetching data for someone's slide deck. We're trying hard to be the first kind. You will ship code that customers touch.

A typical week: Monday stand-up with the team, pick up one feature off the board, pair for the morning, solo for the afternoon, ship by Friday. We review every PR with comments that teach, not gate.

What you'll do.

  • Own small features end-to-end, with a senior engineer pairing on the tricky parts.
  • Write automated tests. You'll learn to love them.
  • Attend one customer call a week (you can be muted; listening is the point).
  • Present one end-of-internship talk on something you learned — technical or domain.
  • If the internship goes well, we talk about a return offer before you leave.

What we're looking for.

  • Currently enrolled, recent grad, bootcamp, or self-taught — we don't care about the pedigree.
  • Some code we can read. A class project is fine if you can explain it.
  • Comfortable asking questions in public Slack channels. Shy-but-emailing is OK too.
  • Can commit to at least 10 weeks at 30+ hours/week, or a full-time summer.

Compensation.

Paid at $35–$45/hr depending on experience. Fully remote, or desk space in Atlanta if you're local. Travel stipend if we bring you out for a team week.

Process.

Intro call (20 min) → one small coding exercise → a technical conversation → offer. We try to get through the whole thing in under a week.
