Overview.

The IQS Flow API is organized around REST. Endpoints accept JSON, return JSON, and use standard HTTP status codes. Base URL below, versioned, and never broken between major versions.

# Production
https://api.iqsflow.com/v2

# EU residency tenant (roadmap, 2026)
# https://api.eu.iqsflow.com/v2
⚑ Independence note
Your API token scopes data to your tenant only. Vendor organizations issued separate client portals do not share a namespace with your data. That's the point.
◉ API status
The v2 API, SDKs, webhooks, and OpenAPI spec documented on this page are rolling out through 2026. Current early-access customers integrate via a private v1 endpoint during onboarding. Contact sales for early-access credentials or the preview specification.

Authentication.

Bearer tokens over HTTPS. Tokens are issued per-service-account under your organization admin, scoped to read or read+write, and can be rotated without downtime.

# Every request includes the bearer
curl https://api.iqsflow.com/v2/inspections \
  -H "Authorization: Bearer iqs_live_sk_••••_SUXgA" \
  -H "Accept: application/json"

Inspections.

The core resource. Every inspection carries a rubric, an evidence set, GPS + timestamps, and a computed score. Create, retrieve, or list against filters.

Create an inspection

POST/v2/inspections
FieldTypeReqDescription
site_idstringrequiredThe site under inspection. See /v2/sites.
frameworkenumrequiredOne of ATA_103, ISSA_CIMS, JCAHO, OSHA_1910, CDC_EVS, ISO_14644, or a custom framework ID.
crew_idstringrequiredThe performing crew. Scoped to the vendor of record.
evidence[]array<Photo>requiredPhotos with EXIF intact. Minimum of one photo per scored item.
notesstringoptionalMax 2,000 characters. Free-text observations.
 
// Request
POST /v2/inspections
Authorization: Bearer iqs_live_sk_••••
Content-Type: application/json

{
  "site_id": "ATL-B14",
  "framework": "ATA_103",
  "crew_id": "SJS-7",
  "evidence": [ /* photos[] with EXIF intact */ ]
}

// Response, 201 Created
{
  "id": "insp_01HS4R2...",
  "score": 94,
  "framework_delta": -1.2,
  "created_at": "2026-03-19T06:01:42Z",
  "status": "scored"
}

Retrieve an inspection

GET/v2/inspections/{id}

Returns the full inspection object, including all evidence URLs (pre-signed, 15-minute expiry), scoring breakdown by item, and the computed framework delta.

Roadmap for 2026. Planned: subscribe to lifecycle events and we POST a signed payload to your endpoint, with exponential-backoff retries for up to 24 hours on failed deliveries.

Event catalog.

The planned event catalog. These events will be emitted once the webhook endpoint ships; subscribe to as many or as few as your integration needs.

inspection.created
A new inspection has been submitted.
inspection.scored
Scoring has completed for an inspection.
inspection.flagged
Inspection fell below configured threshold.
site.score.threshold_crossed
Rolling site score crossed a defined boundary.
vendor.score.updated
Vendor Accountability Score was recalculated.
framework.updated
A framework version was published to the tenant.
reaudit.required
Threshold breach automatically queued a re-audit.
evidence.missing
An item closed without the required evidence.
