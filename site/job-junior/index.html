The role.

We build an inspection and scoring platform used by facilities, healthcare, and aviation operators. The stack is Swift/Kotlin on the capture app, TypeScript + Postgres on the backend, React on the portals. You do not need to know all of it. You need to want to learn all of it.

You'll pair daily with senior engineers. There is no rotation program, no six-month ramp, no "you're not ready for production." Your first merged PR should land in the first week. Your first feature should ship to a customer inside a month.

What you'll do.

  • Pair-program on features across the capture app, scoring backend, and portals.
  • Own small features end-to-end: you spec it, you build it, you ship it, you watch the logs.
  • Shadow at least one real inspection in your first 60 days. Yes, with a clipboard.
  • Write customer-facing release notes. Plain writing is part of the role.
  • Join every customer call you can, silently, to hear what real users say.

What we're looking for.

  • Some real code you can show us. A CS degree is not required; a GitHub we can open is.
  • Comfortable with at least one of: TypeScript, Swift, Kotlin, Python. You'll learn the others.
  • Can debug. Will not just copy-paste the top Stack Overflow answer without reading it.
  • Writes clearly. Async-first teammates need this more than tenured teams do.
  • Curious about the domain. You don't need to love inspections. You do need to want to understand why they exist.

Compensation.

0.5% equity + modest base. We'll talk openly about the full package on the first call. Full-time, remote US or in-office in Atlanta.

Process.

Intro call (30 min) → 90-minute pair-programming session on a small real bug from our tracker → conversation with the founder about the domain → offer. If you already have public code, we may skip the pair session.
